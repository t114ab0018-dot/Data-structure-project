User Story 1：找房租客（學生）
地理位置與條件篩選（對應房源查詢）
作為一位 每天需要通勤上課的在校學生
我想要 透過左右分割畫面，在左側篩選條件（捷運站、房價區間、可否租補開關）進行過濾，並直接在右側「互動式地圖」查看房源位置標記
因為 這樣能幫助我快速過濾出符合預算與捷運通勤需求的物件，節省盲目搜尋與看房的時間。
資訊透明與防踩雷（對應租客房東登入驗證）
作為一位 缺乏租屋經驗的新手租客
我想要 在瀏覽首頁橫向滾動軸的「房源卡片」時，能清楚看見圖片左上角疊加的「可否租補」標籤，以及平台主打的雙向認證「藍勾勾標章」
因為 我擔心遇到假房東或無法申請補助的物件，需要平台透過清楚的 UI 標記為我把關真實性與透明度。
尋找室友（對應室友配對系統）
作為一位 想要找人分租套房以減輕負擔的大學生
我想要 在系統跳轉至配對頁面後，填寫上半部的 10 題生活習慣問卷表單，並在下半部查看系統動態顯示的 3 張「相似習性使用者卡片」與契合度進度條，再點擊「發送聯絡訊息」按鈕
因為 我希望能快速找到作息與習慣相近的好室友，避免日後的合租摩擦。
合約履約與日常維護（對應我的租屋生活）
作為一位 已經順利入住的租客
我想要 在個人後台的網格區塊中，透過「租約期限卡」查看合約內容，在「租金狀態卡」確認本月是否顯示已繳清的綠色標籤，並於「報修追蹤卡」檢視房東處理狀態
因為 我需要一個統一且透明的面板管道來保障我的租屋權益，並確認房東有在處理問題。
User Story 2：房東與包租代管業者
租客篩選與信任建立（對應學生登入）
作為一位 愛惜房屋的房東
我想要 確保平台上配對到的對象，都是必須在表單面板輸入「學號與密碼」完成註冊登入的北科大學生
因為 我希望能將房子租給身分單純、經過學校驗證的優質房客，大幅降低租屋糾紛風險。
合約管理（對應合約簽署頁面）
作為一位 講求效率的房東
我想要 從管理後台發送合約，讓學生在滿版的文件閱讀器模式中瀏覽條文，並於底部固定列勾選同意後點擊「確認並簽約」按鈕
因為 這能大幅省去紙本作業的時間，並透過電子簽署確保合約流程完整。
日常管理與維運（對應房東管理後台）
作為一位 同時管理多間房源的房東
我想要 一登入後台，就能在頂部的四張數據統計卡片中直覺看到「本月收入」與帶有驚嘆號提示的「待處理維修」，並能透過膠囊導覽列快速切換「租金催收」與「修繕請求」列表
因為 我不想在各種通訊軟體中翻找對話紀錄，我需要一個集中化面板來提升管理效率。
User Story 3：平台端（系統管理員 / 校方）
身分與房源審查（對應房東登入與認證頁面）
作為一位 平台系統管理員
我想要 系統強制房東在建立帳號時，必須於指定區塊上傳「身分證」與「房產證明」，並顯示「等待審核通過」狀態攔截其操作
因為 我必須嚴格落實首頁宣告的產權審核機制，將高風險物件阻擋在上架之前。
智能媒合與補助計算（對應房源查詢）
作為一位 負責平台營運的系統管理員
我想要 系統能在背景執行 AI 運算，自動比對學生的預算範圍與通勤需求，並計算租補合格資格，將最適合的 5 張獨立房源卡片推播至橫向滾動軸中
因為 這樣能主動預防學生因資訊落差而錯失優質物件，最大化平台的媒合效率。
糾紛調解追蹤（對應租客及房東資料連動）
作為一位 處理租屋糾紛的校方人員
我想要 平台底層資料庫能夠確實連動學生的「租金/報修狀態卡」與房東的「修繕/催收請求列表」
因為 當糾紛發生時，我需要這些客觀的雙邊數位足跡與狀態標籤（如：處理中 / 已結案）作為協調時的有力證據，確保合約被確實履行。
